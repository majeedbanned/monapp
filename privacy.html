Privacy Policy

This privacy policy applies to the public Monely website and the Monely iPhone app as described on this site. Last updated on March 29, 2026.

Overview

Monely is designed around a local-first finance workflow. Receipt images, expense records, OCR output, and related settings are stored on the device by default. Some features only work by contacting configured services, but those features are specific and product-driven rather than invisible site tracking.

The website inside this `website` folder is a static marketing site. It does not include analytics scripts, ad pixels, or cookie banners by default in the current publish-ready build.

What the app can store

Depending on how you use the app, Monely can store the following information locally on your device:

  • Receipt images and PDFs you import.
  • OCR text extracted from receipts and documents.
  • Expense, income, subscription, and related ledger entries.
  • AI analysis results, classifications, and tags generated from receipt text.
  • Settings such as default currency, payment-source references, profile display preferences, and app lock configuration.
  • Collaboration metadata, shared-ledger state, and member records if you enable collaboration features.
  • Backup files that you choose to create or restore.

When data leaves the device

AI analysis

When you explicitly request AI analysis, extracted receipt text can be sent to the configured analysis endpoint to generate structured results. The app copy already reflects this behavior and this hosted policy keeps that same promise.

Collaboration and sync

If you activate collaboration, create shared ledgers, invite members, or sync shared-ledger changes, relevant collaboration payloads can be exchanged with the configured backend service that supports those features.

Live market data

If you refresh asset prices or fetch market information, requests can be sent to configured market-data providers such as Alpha Vantage and CoinGecko so the app can return pricing and freshness information.

Apple platform services

Camera access, photo-library imports, Face ID, notifications, and in-app purchase flows use Apple platform APIs and services according to the permissions you grant on your device.

How data is used

Monely uses the information above to power the actual app experience: turning receipts into ledger entries, organizing reports, refreshing market data, enabling shared ledgers, and supporting backup or restore flows. The website itself uses data only to deliver static files in the current build.

Your controls

  • You decide when to run AI analysis.
  • You can delete app records and imported items from inside the app.
  • You can enable or disable biometric protection through app settings and device settings.
  • You can choose whether to activate collaboration features.
  • You can control notifications through iOS settings.
  • You can manage backups and restores through the app's backup flows.

Website-only data

This website is currently a static site with no built-in analytics, behavioral tracking, ad pixels, or cookie prompts. If you add analytics, a waitlist form, a helpdesk widget, or similar tools before launch, update this hosted policy before publishing the revised site.

This policy describes the current Monely build as of March 29, 2026. If the app adds new collection, hosting, or third-party integrations later, update this page before distributing the new version.

Policy changes

When Monely's data handling changes in a material way, update this hosted policy and revise the "last updated" date shown on this page. Keep the language aligned with the actual app behavior so the website, App Store metadata, and in-app privacy messaging stay consistent.
